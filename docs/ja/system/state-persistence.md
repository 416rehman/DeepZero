## 永続化サブシステム

パイプラインのデータの整合性は、抽象的な `StateStore` モジュールによってバックアップされます。ファイルシステムにバックアップされる実装は、OS レベルのカーネル割り込み中でも、アトミックなダンプファイルの置換によって破損から保護されます。

### アトミック置換 (`atomic_replace`)

アクティブなアーティファクトに破壊的に書き込むのではなく、永続化レイヤーはシリアライゼーションを中間の `.tmp` バッファにルーティングします。シリアライゼーションが完全に成功した後、`os.replace` によるブロックなしの posix 上書きが行われます。

### ワークスペースの階層

```text
work/<pipeline_name>/
├── run.json             # インスタンスレベルでシリアライズされた状態
├── pipeline.yaml        # 元の YAML 設定のコピー
└── samples/
    └── <sample_id>/     # 分離されたサンドボックスと永続化フォルダ
        └── state.json   # ファイルの永続的な状態
```
