## Архитектура Движка

Движок оркестрации DeepZero работает со строгим направленным графом обработки, определяемым [конвейерами]({{ '/ru/reference/pipeline-yaml.html' | relative_url }}). Механизм выполнения гарантирует отказоустойчивое, возобновляемое [управление состоянием]({{ '/ru/system/state-persistence.html' | relative_url }}).

### Жизненные циклы компонентов

Модель данных проходит через следующие абстракции:

1. **`Sample`**: Базовая структура, выдаваемая процессором `IngestProcessor`. Связывает физический путь с уникальным `sample_id`.
2. **`SampleState`**: Постоянная запись, поддерживаемая `StateStore`. Отслеживает вердикт (`PENDING`, `ACTIVE`, `FILTERED`, `FAILED`, `COMPLETED`).
3. **`ProcessorEntry`**: Фасад, передаваемый процессорам Map/Reduce, использующий отложенную загрузку из памяти.
