## Подсистема Персистентности

Целостность данных обеспечивается модулем `StateStore`. Операции, выполняемые в файловой системе, защищены от повреждения, даже при прерывании на уровне ОС, с помощью атомарной замены файлов.

### Атомарные Замены (`atomic_replace`)

Вместо прямой записи в артефакты (например, `state.json`), уровень персистентности использует временные `.tmp` буферы. После завершения сериализации происходит принудительная перезапись через `os.replace`.

### Иерархия Рабочего Пространства

```text
work/<pipeline_name>/
├── run.json             # Сериализованное состояние
├── pipeline.yaml        # Снимок YAML конфигурации
└── samples/
    └── <sample_id>/     # Изолированная песочница
        └── state.json   # Сериализованное состояние образца
```
